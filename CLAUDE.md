## Overview

This is a personal GitHub profile repository that displays developer activity and statistics. It is **not a traditional software project** - it's a specialized profile page repository that uses GitHub Actions for automated content generation.

## Architecture

### README.md Structure
The profile README contains:
- Personal introduction and interests
- **Dynamic activity section** (lines 9-12) - Auto-updated by GitHub Actions
- SVG stat cards generated by external actions

### GitHub Actions Workflow

**File**: `.github/workflows/update-readme.yml`

**Triggers**:
- Scheduled: Daily at midnight UTC (`cron: '0 0 * * *'`)
- Manual: `workflow_dispatch` allows manual triggering from GitHub UI

**Workflow Steps**:
1. **Generate Activity** (jamesgeorge007/github-activity-readme@master)
   - Updates the `<!--START_SECTION:activity-->` to `<!--END_SECTION:activity-->` block in README.md
   - Shows recent GitHub activity across all repositories
   - Configured: `MAX_LINES: 10`

2. **Generate Stats Cards** (readme-tools/github-readme-stats-action@v1)
   - Creates `profile/stats.svg` - Overall GitHub statistics
   - Creates `profile/top-langs.svg` - Programming language distribution
   - Theme: `material-palenight`
   - Options: `include_all_commits=false`, `hide_title=true`, `show_icons=true`

3. **Commit Changes**
   - Bot commits as `github-actions[bot]`
   - Commits both SVG files and updated README.md
   - Uses `|| exit 0` to gracefully handle no-change scenarios

## Important Notes

### Content Generation is External
- Activity data and stats cards are **generated by GitHub Actions**, not local tooling
- Do not manually edit the `<!--START_SECTION:activity-->` block
- Do not manually modify SVG files in `profile/`

### Workflow Commit Messages
The automated workflow uses a standardized commit message:
```
⚡ Update README with recent activity
```

### Theme Configuration
Visual cards use the `material-palenight` theme. When modifying workflow options, ensure consistent theming.

### Activity Limits
The activity feed is capped at `MAX_LINES: 10`. Adjust this in the workflow YAML if more/fewer activities should display.
